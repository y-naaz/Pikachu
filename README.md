# ⚡ Pikachu

**Your personal engineering memory.**

Software engineers forget things. You learn a concept on Monday, struggle with a bug on Wednesday, finally understand how CORS works on Friday — and by next month it's gone. Pikachu fixes that.

Pikachu is a local-first knowledge base where you **capture, organize, and retain** everything you learn as an engineer. Paste some code, ask a question, or describe a concept — Pikachu generates a structured learning entry, stores it in your local database, and helps you **never forget it** through spaced repetition.

## The Problem

- You learn things during coding sessions but never write them down
- Weeks later you face the same problem and can't remember the solution
- Your knowledge lives in scattered notes, Slack messages, and browser tabs
- There's no system to help you **retain** what you've learned

## How Pikachu Works

### 1. Capture — Explain & Save

Paste any code snippet, documentation, or question into Pikachu. The AI (Claude or OpenCode) generates a structured learning entry with:

- **What** it is
- **Why** it exists
- **How** it works (with concrete detail)
- **Summary** (one-liner for quick recall)
- **Concepts** and **related concepts** for connecting ideas

You can also add learnings manually or capture them from your VS Code editor.

### 2. Organize — Knowledge Explorer

Your learnings are organized by concept. The Knowledge Explorer shows:

- All your concepts as browsable cards
- How concepts connect to each other
- Which concepts have learnings due for review
- Language-based filtering and full-text search

Click any concept to see all its learnings, connected concepts, and suggested next topics.

### 3. Retain — Spaced Repetition

Pikachu uses the **SM-2 algorithm** (the same one behind SuperMemo and Anki) to schedule reviews at optimal intervals:

- Review a learning → rate how well you remembered (0-5)
- Pikachu calculates the next review date based on your performance
- Easy recall → review again in weeks/months
- Struggled → review again tomorrow
- Over time, you build lasting knowledge

### 4. Search — Full-Text Search

Find anything instantly with SQLite FTS5-powered search. Search across titles, explanations, code snippets, and concepts. Type a prefix and get instant results.

## Stack

| Layer | Technology |
|-------|-----------|
| Frontend | Next.js 16 (App Router) + TypeScript + Tailwind v4 |
| Database | SQLite via Prisma ORM |
| Search | SQLite FTS5 virtual tables |
| AI Backend | Claude API (`@anthropic-ai/sdk`) or OpenCode CLI |
| Spaced Repetition | SM-2 algorithm (custom implementation) |
| VS Code Extension | TypeScript, webview panels |

## Getting Started

```bash
# 1. Install dependencies
npm install

# 2. Apply database migrations
npm run db:migrate

# 3. Run the dev server
npm run dev      # http://localhost:3001
```

### AI Backend Setup

Pikachu supports two local AI backends — no cloud API key required:

| Backend | CLI | How to authenticate |
|---------|-----|-------------------|
| **Claude Code** (default) | `claude` | `claude login` |
| **OpenCode** | `opencode` | `opencode providers` |

Switch between them on the Explain & Save page.

### VS Code Extension

```bash
cd vscode-extension
npm install && npm run compile
# Press F5 in VS Code to open Extension Development Host
```

Commands: **Pikachu: Explain Selection** · **Pikachu: Explain File** · **Pikachu: Search Learnings**

## Features

### Dashboard
Your engineering memory at a glance — total learnings, top concepts, learning streak, and recent entries.

### Explain & Save
The core feature. Paste code or a question, get a structured learning entry generated by AI. Supports streaming for instant feedback.

### Learnings
Browse, search, and manage all your learnings. Each learning shows its explanation in rich markdown with syntax-highlighted code blocks.

### Knowledge Explorer
Learnings grouped by concept. Browse what you know, search across concepts, and discover connections between ideas.

### Review
Spaced repetition review sessions. See which learnings are due, recall the answer, and rate your performance. Pikachu schedules the next review based on how well you remembered.

### Search
Full-text search powered by SQLite FTS5. Finds matches across titles, explanations, code snippets, and concepts with prefix matching.

### Learning Details
Rich markdown rendering with syntax highlighting for code blocks. Shows explanation, related concepts, repository context, and review status.

## How the Data Model Works

The `Learning` model stores everything about a captured concept:

| Field | Purpose |
|-------|---------|
| `title` | Concise name for the concept |
| `explanation` | Full markdown explanation (What/Why/How) |
| `summary` | One-line recap for list views |
| `codeSnippet` | Key code or diff |
| `concepts` | JSON array of core concepts |
| `relatedConcepts` | JSON array of adjacent topics |
| `language` | Primary language/technology |
| `repository` | Source repo name |
| `sourceType` | Where it came from (manual/claude/vscode) |
| `nextReview` | When the next spaced repetition review is due |
| `reviewCount` | How many times reviewed |
| `easeFactor` | SM-2 ease factor (starts at 2.5) |
| `interval` | Current interval in days |

**Note:** `concepts`, `relatedConcepts`, and `tags` are stored as JSON strings because SQLite has no array type. Conversion happens in `src/lib/learning.ts`.
